### 客户概况

温州市国彩真空科技有限公司是浙南地区规模最大的PVD镀膜加工服务商之一，年加工能力约200万件，客户分布在渔具、餐具、五金、3C配件等多个行业。公司采用B2B来料加工模式，按件收费，打样周期4-7天。

总经理李进汉技术出身，对工艺细节和实际回报敏感，决策风格直接。当前公司60人规模，技术团队约8人、品管团队约4人，无专职IT团队，数字化基础为零，决策者对新技术有基本认知。

## 2. 解决方案

### 整体思路

PVD来料加工企业的核心困境：订单靠微信催，批次出问题查不出原因，工艺能力靠嘴说看不见。整个过程缺乏透明度，出了问题只能靠经验猜。

本方案以SaaS为载体，将订单从进到出的全流程串起来，每道工序扫码留记录，质量问题可快速溯源。同时，打造PVD工艺能力与案例展示平台，把看不见的工艺优势变成客户能直接看到的图文内容，降低销售沟通成本，帮助获客。

整体分三期推进：一期聚焦订单和追溯，解决最痛的两个问题快速验证价值；二期扩展到生产和设备，打通车间数据；三期加上客户门户和成本分析，同时将展示平台纳入整体获客体系。

> **注**：置信度为推演的功能模块，其具体字段和流程需在一期需求调研中与客户现场确认后再锁定；展示平台为已确认需求，需在一期优先确认内容方向与部署形式。

### 功能设计

| 功能 | 解决的问题 | 业务价值 |
|------|-----------|---------|
| 订单管理 | 工单从排程、镀膜、质检到出货各环节状态实时可见，订单优先级可设置和调整 | 客户减少催单，跟单员减少重复答复，出货节点可控 |
| 质检记录与批次追溯 | 批次关联工艺参数、设备编号、操作人员，出现质量问题时可快速定位。追溯字段范围（具体包含哪些工艺参数）待一期现场确认后配置 | 减少批量事故损失，提升客诉处理效率 |
| 扫码追溯 | 每批次/工件绑定二维码，操作人员扫码完成报工和质检记录录入，兼容手机扫码。标签方案（现有条码复用或新建二维码）待现场评估后确定 | 降低一线人员操作门槛，纸质记录习惯可平滑过渡 |
| 生产进度看板 | 日/周视图展示各工单当前工序和预计完成时间，班组长快速识别延误订单 | 主动协调资源，减少被动救火 |
| 设备稼动率监控 | 记录各真空镀膜设备启停时间、运行时长、故障记录，支撑OEE计算 | 识别设备利用率瓶颈，支撑预防性维护 |
| 客户自助查询门户 | 客户通过工单链接查看自己订单当前状态，减少跟单员重复答复 | 提升客户体验，释放跟单工作量 |
| 打样与交期管理 | 打样工单与量产工单分开管理，打样周期预警，量产交期与打样结果挂钩 | 打样进度可控，量产交期承诺有据 |
| 成本核算 | 按订单分工艺类型统计加工费，自动核算毛利 | 快速出具订单成本和利润数据，支撑报价决策 |
| **PVD镀膜能力与案例展示平台** | **无对外展示渠道，客户对工艺能力缺乏直观认知，销售靠嘴说效率低** | **客户点击即看明白工艺效果，减少销售沟通成本，辅助获客转化** |

展示平台功能说明（需与客户确认内容方向后细化）：

- **镀膜工艺展示**：不同工艺的效果对比图文，按基材类型筛选。具体工艺类型（离子镀/磁控溅射/UV电镀等是否全部覆盖）待与客户确认实际可展示范围
- **行业应用案例**：渔具、餐具、五金、3C配件等典型行业的合作案例，含产品图、镀膜类型和效果描述
- **工艺能力介绍**：设备类型和数量、可加工尺寸范围、可镀颜色和膜厚范围等参数化展示
- **产品图册**：合作客户产品展示，支持分类检索
- **技术参数页面**：常见工艺问题解答，建立专业认知壁垒

展示平台部署形式（独立部署或与官网整合）需与客户确认现有官网情况后确定。内容管理后台支持运营人员自主更新案例和图片，无需技术介入。

### 技术方案

| 技术维度 | 方案 | 备注 |
|----------|------|------|
| 部署方式 | SaaS多租户架构，按企业订阅，无需本地服务器和维护 | 无IT团队，适合 |
| 前端交互 | Web端管理后台 + 移动端H5（手机扫码即用，无需安装App） | 需一期现场评估车间网络覆盖情况 |
| 数据采集 | 条码/二维码标签，兼容普通智能手机扫码。PDA采购确有必要时在一期评估后再定 | 一期优先验证手机扫码可行性，避免硬件先行投入 |
| 数据存储 | 云端加密存储，批次追溯数据按监管要求保留 | |
| 集成能力 | 一期不包含系统对接；如后续有与财务/CRM打通需求，可通过开放API实现 | 客户暂无IT能力，一期不承诺集成 |
| 展示平台 | 独立部署或与官网集成，响应式设计，支持PC和移动端访问，内容管理后台供运营自主维护 | 部署形式待客户确认官网现状后确定 |
| 设备数据采集 | 一期以人工记录为主；如现场确有设备自动化采集需求，可进一步探讨PLC接口对接方案，但需评估设备接口开放情况 | 不在一期承诺 |

### 差异化优势

| 对比维度 | 现状竞对（Excel+电话+微信） | 通用ERP（用友/金蝶/管家婆） | 本方案 |
|----------|---------------------------|---------------------------|--------|
| 定位 | 无系统支撑 | 各行业通用 | 小型PVD镀膜加工企业专属 |
| 核心能力 | 无 | 大而全 | 订单可视化+批次追溯+工艺展示 |
| 部署复杂度 | — | 高，配置复杂 | 低，SaaS开箱即用 |
| 实施周期 | — | 数月 | 一期4-6周起，快速验证价值 |
| IT要求 | 无 | 需专人维护 | 无需专职IT |
| 对外展示 | 无 | 无 | 配套工艺展示平台，辅助获客 |

## 3. 实施路径

### 阶段概览

| 阶段 | 周期（参考） | 核心目标 | 关键交付物 |
|------|------------|----------|-----------|
| 启动 | 约1-2周 | 需求确认，数据准备，环境搭建 | 需求确认备忘（含追溯字段清单确认），基础数据配置完成 |
| 一期 | 约4-6周 | 订单管理+批次追溯+质检记录+扫码追溯 | 工单全流程在线，批次质量问题可追溯。本阶段为关键验证期，需确保扫码追溯在实际车间环境中可用 |
| 二期 | 约3-4周 | 生产看板+设备运行时长+基础OEE | 车间进度可视化，设备利用率数据可见 |
| 三期 | 约4-6周 | 客户门户+成本核算+展示平台+报表分析 | 客户自助查进度，订单成本透明，工艺展示平台上线 |

> 注：各阶段周期为参考值，实际周期需根据现场需求确认情况调整。

展示平台（已确认需求）在一期内完成需求调研和原型确认，与三期客户门户同步上线。展示平台的内容填充（案例拍摄、图文整理）可与系统开发并行推进，需客户安排专人负责素材准备——如暂无专人，内容补充可在三期上线后继续完善，不影响系统交付。

### 关键里程碑

| 时间节点 | 里程碑 |
|----------|--------|
| 第1-2周 | 需求确认，签订合同，完成基础数据配置（工艺类型、设备列表、人员权限），确认扫码标签方案和车间网络条件 |
| 第3-6周 | 一期功能开发完成，先以1-2条真实工单在单台设备上试点运行，确认扫码流程可行、数据可信后再扩大范围 |
| 第7-10周 | 二期功能上线 |
| 第11-16周 | 三期功能上线，展示平台同步发布，完整方案交付 |

> **试运行策略说明**：一期试运行建议从小范围开始——选取单台设备，1-2条工单先行验证，纸质记录与系统并行运行2周，确认流程跑通、数据可信后再全面推广。零数字化基础企业全面铺开的风险较高，小步验证可有效降低推进阻力，也更符合李总对"实际回报"的关注。

## 4. 成本与预算

> 本节为便于客户决策参考，实际价格需商务阶段确认。

### 订阅费用（持续成本）

本方案采用SaaS按企业订阅模式，按期收费，支持按需开通模块。

| 模块分组 | 包含功能 | 说明 |
|----------|---------|------|
| 核心模块（一期） | 订单管理、质检记录、扫码追溯 | 一期必开 |
| 生产模块（二期） | 生产看板、设备管理 | 一期后按需开通 |
| 运营模块（三期） | 客户门户、成本核算、报表分析 | 按需开通 |
| 展示平台 | 工艺展示、案例图册等 | 独立部署时费用另计 |

具体订阅价格需商务阶段确认。建议客户一期先开通核心模块，验证价值后再逐步扩展，控制初期投入。

### 硬件投入（一次性成本，需客户自购）

| 项目 | 说明 | 是否必须 |
|------|------|---------|
| 条码/二维码标签 | 每批次/工件打印一张，建议使用普通标签纸配合现有打印机 | 必须，按需采购 |
| 扫码标签打印工具 | 标签模板软件；如客户已有打印机，可直接使用 | 可选 |
| PDA手持设备 | 一期优先用手机扫码，确有必要再采购；如需采购预算另计 | 一期非必须 |
| 车间网络覆盖 | 如车间WiFi盲区较多，可能需要网络优化投入 | 需一期现场评估 |
