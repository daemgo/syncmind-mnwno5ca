### 客户概况

温州市国彩真空科技有限公司是一家PVD真空镀膜加工服务商，年加工能力约200万件，客户分布在五金、餐具、渔具、3C配件等行业。来料加工、按件收费是主要模式，打样周期4-7天。

公司创始人李进汉是技术出身，拥有多项专利。拍板靠他，对技术细节和实际回报敏感，不喜欢虚词。

### 当前现状与挑战

| 现状 | 带来的挑战 |
|------|------------|
| 订单跟踪靠电话和微信沟通 | 订单状态不透明，客户频繁询问进度，跟单员重复答复 |
| 批次质量问题追溯困难 | 一旦出现镀膜色差、附着力不足等问题，无法快速定位工艺参数、设备和操作人员，返工成本高 |
| 排产排程依赖Excel | 无法直观看到设备负载和工序冲突，人工排产容易出现疏漏 |
| 无系统支撑 | 来料→排程→镀膜→质检→出货全链路无数据沉淀，无法形成可分析的经营报表 |

### 核心需求

**P0 优先级**

| 需求 | 对应业务场景 |
|------|-------------|
| 订单全流程可视化 | 客户来料后创建工单，工单从排程、镀膜、质检到出货各环节状态实时可见，减少电话微信问进度（待首访确认痛点强度） |
| 镀膜工艺质量追溯 | 每批次关联工艺参数（真空度、靶材、气体流量、镀膜时间）、设备编号、操作人员，出现质量问题时可快速溯源（待首访确认追溯深度需求） |
| 打样周期与交期管理 | 打样与量产分开管理，打样4-7天自动预警，量产交期与打样结果挂钩（待确认打样量产是否已分开管理） |

**P1 优先级**

| 需求 | 对应业务场景 |
|------|-------------|
| 真空镀膜设备稼动率监控 | 设备启停成本高，实时记录设备运行时长和故障停机，支持OEE计算（待确认是否存在设备故障影响交付记录） |
| 按件计费成本核算 | 同一工件可能涉及离子镀、磁控溅射、UV电镀等多种工艺，需按订单分工艺核算加工费和利润（待确认当前成本核算方式） |

**P2 优先级**

| 需求 | 对应业务场景 |
|------|-------------|
| 客户自助查询门户 | 客户扫码或通过链接查看自己工单进度，减少跟单员重复答复（需评估B端客户接受度） |
| 生产进度看板 | 班组长在车间通过看板查看日/周生产进度，快速识别延误工单（需了解班组长的实际查看习惯） |

## 2. 解决方案

### 整体思路

PVD来料加工最常见的问题：订单靠微信催，批次出问题查不出原因。整个过程缺乏透明度，出了问题只能靠经验猜。

本方案用SaaS把订单从进到出的全流程串起来，每道工序扫码留记录。出了问题能快速溯源到设备、批次、操作人员。客户自己查进度，减少跟单员重复答复。第一期先做订单和追溯两个核心模块，快速验证价值，后续再逐步扩展到设备管理和成本分析。

### 功能设计

| 功能模块 | 功能 | 解决的问题 | 业务价值 |
|----------|------|------------|----------|
| 订单管理 | 工单创建、状态流转（待排程→镀膜中→质检中→待出货→已完成）、优先级设置 | 订单状态不透明，客户频繁电话微信问进度 | 客户减少催单，跟单员减少重复答复 |
| 质检记录与批次追溯 | 批次编码关联工艺参数（真空度、靶材、气体流量、镀膜时间）、设备编号、操作人员 | 质量问题发生后追溯困难，返工成本高 | 质量问题可快速定位责任人，减少批量事故损失 |
| 扫码追溯 | 每批次/工件绑定二维码或条码，支持PDA或手机扫码查看状态和追溯信息 | 操作工人习惯纸质记录，需要低门槛数据采集方式 | 扫码即记录，降低一线人员操作成本 |
| 生产进度看板 | 日/周视图，班组长在车间大屏或手机端查看各工单进度 | 排产靠Excel，无法直观看到进度和延误 | 班组长快速识别延误工单，主动调整排程 |
| 设备稼动率监控 | 记录各真空镀膜设备启停时间、运行时长、故障记录 | 设备启停成本高，故障影响交付但无数据支撑决策 | 支撑OEE计算，识别设备利用率瓶颈 |
| 客户自助查询 | 客户通过工单链接或扫码查看订单当前状态 | 客户反复问进度，跟单员重复答复同一问题 | 减少跟单工作量，客户体验提升 |
| 打样与交期管理 | 打样工单与量产工单分开管理，打样周期4-7天自动预警，量产交期自动计算 | 打样结果影响量产交期，但两者管理混在一起看不清 | 打样进度可控，量产交期承诺有依据 |
| 成本核算 | 按订单分工艺类型统计加工费，自动核算毛利 | 按件计费，工艺组合多样，人工核算费时且易错 | 快速出具订单成本和利润数据，支撑报价决策 |

### 技术方案

| 技术维度 | 方案 |
|----------|------|
| 部署方式 | SaaS多租户架构，按企业订阅，无需本地服务器和维护 |
| 前端交互 | Web端管理后台 + 移动端H5（手机扫码即用，无需安装App） |
| 数据采集 | 条码/二维码标签，兼容PDA手持设备和普通智能手机扫码 |
| 数据存储 | 云端加密存储，批次追溯数据按监管要求保留 |
| 集成能力 | 开放API，支持与客户现有财务系统或CRM对接（如有） |
| 设备数据采集 | 一期以人工记录为主；如现场确有设备自动化采集需求，可进一步探讨PLC接口对接方案 |

### 差异化优势

| 对比维度 | 通用ERP（用友/金蝶/管家婆） | 本方案 |
|----------|---------------------------|--------|
| 定位 | 各行业通用 | 小型PVD镀膜加工企业专属 |
| 部署复杂度 | 高，配置复杂 | 低，SaaS开箱即用 |
| 核心功能 | 大而全，小企业用不上大部分功能 | 轻，订单可视化+批次追溯精准 |
| 实施周期 | 数月 | 4-6周起，快速验证价值 |
| IT要求 | 需专人维护 | 无需专职IT |

## 3. 实施路径

### 阶段概览

| 阶段 | 周期（参考） | 核心内容 | 上线目标 |
|------|------------|----------|----------|
| 一期 | 约4-6周 | 订单管理 + 质检记录与批次追溯 + 扫码追溯基础 | 工单全流程在线，批次质量问题可追溯，解决核心痛点 |
| 二期 | 约3-4周 | 生产进度看板 + 设备运行时长记录 + 基础OEE | 班组长随时掌握车间进度，设备利用率数据可见 |
| 三期 | 约3-4周 | 客户自助查询门户 + 成本核算模块 + 报表分析 | 客户自助查进度减少催单，订单成本和利润数据支撑经营决策 |

> 注：各阶段周期为参考值，实际周期需根据现场需求确认情况调整。

### 关键里程碑

| 时间节点 | 里程碑 |
|----------|--------|
| 第1周 | 需求确认，签订合同，完成基础数据配置（工艺类型、设备列表、人员权限） |
| 第2-3周 | 一期功能开发完成，选取2-3条真实工单试运行 |
| 第4-6周 | 一期正式上线，操作人员培训，反馈迭代 |
| 第7-10周 | 二期功能上线 |
| 第11-14周 | 三期功能上线，完整方案交付 |

## 4. 风险与下一步

### 风险表

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| 操作人员抵触数字化，沿用纸质记录 | 中 | 高 | 一期设计极简操作流程（扫码即记录），配合1-2次现场培训，先试点后推广 |
| 扫码追溯依赖条码/二维码体系，现场标签缺失或不规范 | 中 | 高 | 行动3中专项确认扫码标签方案，必要时提供标签打印工具和模板 |
| 设备数据采集需人工录入，准确性难以保障 | 中 | 中 | 一期以人工记录为主；如确有需求，可探讨PLC接口方案，但需评估设备接口开放情况 |
| 打样与量产混线管理，流程梳理复杂 | 中 | 中 | 一期对打样和量产工单分别建立独立状态机，上线前与车间确认分线规则 |
| 预算有限，需控制订阅成本 | 中 | 中 | SaaS按需订阅模式，一期仅开通核心模块，后续按需扩展 |
| 追溯字段配置依赖现有工艺记录方式，若现有记录不完整，追溯深度受限 | 低 | 中 | 行动2中要求客户提供现有工艺记录表，据此确定追溯字段优先级，优先上线关键参数 |

### 下一步行动表

| 序号 | 行动 | 负责方 | 目标产出 |
|------|------|--------|----------|
| 1 | 安排一次现场或线上沟通，确认一期核心需求细节和车间操作流程 | 双方 | 明确一期功能边界，输出需求确认备忘 |
| 2 | 提供车间现有工单模板、工艺参数记录表、质检记录表等参考材料 | 客户 | 用于系统字段配置和默认值设计（追溯字段的完整度取决于现有记录基础） |
| 3 | 确认扫码标签方案（现有条码/新建二维码），以及车间网络条件 | 双方 | 确定扫码追溯实现方式，评估PDA或手机扫码的适用性 |
| 4 | 商务条款确认，签订合同，启动实施 | 双方 | 合同签订 |
